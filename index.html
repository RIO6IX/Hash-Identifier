How detection works

Signals used by the app

Modern password hashes often self-identify using prefixes such as $argon2id$ or $2b$. Older fast hashes such as MD5 and SHA-1 usually expose only length and character set, so the app returns ranked candidates instead of pretending there is one certain answer.

1. Prefix Argon2, bcrypt, scrypt, Unix crypt, phpass, Django, LDAP.
2. Known layout MySQL5, NetNTLMv1/v2, DES crypt, JWT, URL, base encodings.
3. Hex length 32, 40, 56, 64, 96, 128 and other common digest sizes.
4. Safety note This identifies formats and suggests commands for authorized testing only.
